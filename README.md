# Действия управления пользователями

### Создание пользователя

Метод `actionCreate` отвечает за создание новой модели `User`.

- **Процесс**:
  1. Создает новый объект `User`.
  2. Если запрос является POST-запросом:
     - Загружает данные в модель.
     - Сохраняет модель в базу данных.
     - Перенаправляет на страницу 'view' при успешном сохранении.
  3. Если это не POST-запрос, загружает в модель значения по умолчанию.
  4. Отображает представление 'create' с моделью.

### Обновление пользователя

Метод `actionUpdate` обновляет существующую модель `User`.

- **Параметры**:
  - `$id`: ID модели `User`, которую нужно обновить.

- **Процесс**:
  1. Находит существующую модель `User` с использованием переданного ID.
  2. Если запрос является POST-запросом:
     - Загружает данные в модель.
     - Сохраняет изменения в базе данных.
     - Перенаправляет на страницу 'view' при успешном сохранении.
  3. Отображает представление 'update' с моделью.

- **Исключения**:
  - Выбрасывает `NotFoundHttpException`, если модель не может быть найдена.

### Удаление пользователя

Метод `actionDelete` удаляет существующую модель `User`.

- **Параметры**:
  - `$id`: ID модели `User`, которую нужно удалить.

- **Процесс**:
  1. Находит и удаляет модель `User` с использованием переданного ID.
  2. Перенаправляет на страницу 'index' после успешного удаления.

- **Исключения**:
  - Выбрасывает `NotFoundHttpException`, если модель не может быть найдена.

### Поиск модели

Метод `findModel` извлекает модель `User` на основе значения ее первичного ключа.

- **Параметры**:
  - `$id`: ID модели `User`, которую нужно найти.

- **Процесс**:
  - Ищет модель с использованием переданного ID.
  - Возвращает модель, если она найдена.

- **Исключения**:
  - Выбрасывает `NotFoundHttpException` с сообщением "The requested page does not exist.", если модель не может быть найдена.
